## Scope

Track the concrete follow-up work that comes out of the whole-project code review.
This epic should only contain agreed fixes or refactors, not every review observation.

## Acceptance Criteria

- [ ] Each accepted review finding is captured as a focused Bears task.
- [ ] Tasks are small enough to implement in one focused session or a short sequence of dependent sessions.
- [ ] Each task includes clear acceptance criteria, implementation notes, and testing expectations.
- [ ] The epic can be closed by completing the agreed child tasks without extra coordination work.
